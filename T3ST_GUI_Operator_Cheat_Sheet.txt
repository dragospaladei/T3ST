Purpose
- Edit simulation parameters
- Load scenario presets
- Configure up to 3 parameter sweeps
- Export simulation runs to .dat files

Required Files
- parameters.yaml
- scenarios.yaml
Located either in ./config/ or next to Define_Simulations.py

Typical Workflow
1. Set Simulation number
2. Select Scenario
3. Edit parameters if needed
4. Configure sweeps (optional)
5. Preview sweeps
6. Export parameters

Parameter Rules
- Scientific notation allowed
- All parameters must be filled before export
- Integers must be integer-like
- Enums must be selected
- Checkboxes export 0/1

Scenarios
- Dictionary format recommended (keyed by parameter name)
- Legacy list format supported
- Startup warnings indicate typos or length mismatches

Sweeps
- Up to 3
- Only int/float parameters
- No duplicates
- Cartesian product expansion

Export Format
<total_rows>
<num_parameters>
p1 p2 p3 ...
v1 v2 v3 ...

Loading
- Header optional
- First data row loaded into GUI
